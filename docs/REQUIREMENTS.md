## 1. はじめに

本ドキュメントは、子どものトイレトレーニングを支援する新規モバイルアプリケーション「（仮称）トイレの冒険」の開発に関する要件を定義するものです。本アプリは、iOSおよびAndroidの両プラットフォームに対応します。本ドキュメントは、プロジェクト関係者間の共通理解を形成し、開発の指針とすることを目的とします。

## 2. プロジェクトの背景と目的

### 2.1. 背景

トイレトレーニングは、多くの保護者が直面する育児における重要なマイルストーンですが、子供の抵抗や保護者の孤独感により、多くの家庭で困難な課題となっています。既存のトイレトレーニングアプリは、保護者向けの単なる記録ツールか、子供向けのエンターテイメントのどちらかに偏っており、親子が一体となって楽しく取り組めるソリューションが不足しています。特に日本市場においては、高品質な日本語ネイティブアプリが不在であり、ソーシャル機能を通じて保護者同士が繋がれる場も存在しません。

### 2.2. 目的

本プロジェクトの目的は、これらの市場ギャップを埋める、革新的なトイレトレーニングアプリを開発することです。具体的には、以下の3点を達成します。

1.  **親子の協働体験の創出**: 保護者の記録と子供のご褒美システムを連動させ、親子が同じ目標に向かって楽しく取り組める体験を提供します。
2.  **保護者の孤独感の解消**: 匿名のソーシャルタイムライン機能を通じて、他の家庭の様子を知り、応援し合えるコミュニティを形成します。
3.  **子供の自発的な動機付け**: 心理学に基づいた段階的なご褒美システムと魅力的なキャラクター演出により、子供が自らトイレに行きたくなるような内発的動機付けを促します。

## 3. 市場・競合分析

### 3.1. 市場のギャップ

iOS App Storeの競合9アプリを分析した結果、以下の4つの明確な市場ギャップが確認されました。

- **親子統合型アプリの不在**: 保護者向け管理機能と子供向けエンタメ機能が高レベルで統合されたアプリが存在しない。
- **ソーシャル機能の欠如**: リアルタイムで利用者同士が繋がれるタイムライン型のソーシャル機能が皆無。
- **高品質な日本語アプリの不在**: 既存の日本語アプリは品質やUXに課題がある。
- **本格的なご褒美システムの不足**: 長期的なモチベーションを維持する段階的なご褒美サイクルを持つアプリがない。

### 3.2. 主要競合

| 競合アプリ | 強み | 弱み |
| :--- | :--- | :--- |
| **Potty Whiz** | 詳細なトラッキング機能、Apple Watch連携 | 子供向け要素なし、ソーシャル機能なし |
| **Go Time** | 子供向けエンタメ性、完全無料 | 記録・分析機能なし、ソーシャル機能なし |
| **トイトレ（ムーニー）** | AR機能、発達心理学に基づく設計 | 専用グッズ必須、バグが多い、評価が低い |

## 4. アプリのコンセプトと差別化戦略

### 4.1. コアコンセプト

**「親子で挑む、ご褒美いっぱいのトイレ大冒険」**

トイレトレーニングを「親子で協力してクリアしていく冒険」と再定義します。保護者の記録が子供の冒険の進行に繋がり、子供の成功が家族全体の喜びとなる、ポジティブなサイクルを創出します。

### 4.2. 差別化戦略

本アプリは、**「親子統合型 × ソーシャル × ゲーミフィケーション × 日本語ネイティブ」**という独自のポジショニングで、以下の5点で競合と明確に差別化します。

1.  **親子統合型アプリ**: 保護者の記録と子供のご褒美をシームレスに連動。
2.  **ソーシャルタイムライン**: 匿名の「うんち報告」と「応援スタンプ」で孤独感を解消。
3.  **段階的ご褒美コレクション**: シールを集めて特別なキャラクターやアニメを解放する長期的な目標設定。
4.  **日本語ネイティブ・日本市場特化**: 日本の育児文化に最適化された高品質なUXを提供。
5.  **トイレ促進キャラクター演出**: 子供が自発的にトイレに向かいたくなる仕掛け。

## 5. ターゲットユーザー

- **ペルソナ**: 1歳半〜3歳の子供を持つ、初めてのトイレトレーニングに不安や負担を感じている日本の保護者。
- **ニーズ**: 子供に楽しくトイレを学んでほしい。他の家庭の進捗を知りたい。トレーニングの進捗を簡単に管理したい。

## 6. 機能要件（MVPスコープ）

ミニマムスタート（MVP）として、以下の4つのコア機能に絞って開発を行います。

### 6.1. ユーザープロフィール機能 (ID: F-001)

- **概要**: ユーザー（子供）の基本的な情報を登録・管理する。
- **詳細**:
    - ニックネーム（10文字以内）を登録できる。
    - 性別（男の子/女の子/設定しない）を選択できる。
    - 年齢（月齢）を設定できる。
    - 複数の選択肢から基本となる「アバター」を選択できる。
    - プロフィール情報はアプリ内で変更可能である。

### 6.2. トイレ記録機能 (ID: F-002)

- **概要**: 子供のトイレ活動をワンタップで簡単に記録する。
- **詳細**:
    - ホーム画面に「便座にすわれた」「おしっこできた」「うんちできた」の3つの記録ボタンを配置する。
    - 各ボタンをタップすると、タイムスタンプと共に活動が記録される。
    - 記録と同時に、後述のご褒美システムが作動する。

### 6.3. ご褒美システム (ID: F-003)

- **概要**- 概要: トイレ記録に応じてシールを付与し、アバターの着せ替えアイテムを報酬として提供することで、子供のモチベーションを高める。
- 詳細:
    - トイレ記録の種類に応じて、異なる枚数のシールが付与される（例: 座れた→1枚, おしっこ→2枚, うんち→3枚）。
    - 獲得したシールは「シール帳」画面に貼られていく。
    - シール帳が一杯になると（規定枚数に達すると）、ご褒美としてアバターが着せ替えられる「服装」や「アクセサリー」などのアイテムがランダムで付与される。
    - 付与されるアイテムは、設定された性別に応じたものが提供される。
    - トレーニング初期は、少ない枚数（例: 5枚）でシール帳が一杯になるように設定し、達成感を頻繁に得られるようにする。
    - ご褒美獲得時には、特別な祝福アニメーションを表示する。

### 6.4. ソーシャルタイムライン機能 (ID: F-004)

- **概要**: 匿名の成功報告を共有し、ユーザー同士で応援し合える場を提供する。
- **詳細**:
    - 他のユーザーの「（ニックネーム）がトイレでできました！」といった成功記録が時系列で表示される。
    - 各投稿には、複数の「応援スタンプ」（例: 「すごい！」「がんばったね！」）を送ることができる。
    - 投稿は匿名で行われ、個人を特定する情報は表示されない。
    - タイムラインの閲覧は誰でも可能だが、投稿はトイレ記録と連動して自動的に行われる（ON/OFF設定可能）。

## 7. 非機能要件

| 項目 | 要件 |
| :--- | :--- |
| **パフォーマンス** | 全ての画面遷移、操作は2秒以内に応答を返すこと。 |
| **ユーザビリティ** | 幼児と保護者の両方が、直感的に操作できるシンプルなUI/UXであること。 |
| **セキュリティ** | ユーザーデータは安全に管理し、個人情報保護法を遵守すること。 |
| **互換性** | iOS 16.0以降、およびAndroid 10.0以降に対応すること。 |

## 8. 収益化戦略

MVP段階では収益化機能は実装しない。Phase 2以降で、以下のフリーミアムモデルの導入を検討する。

- **無料プラン**: 基本機能（トイレ記録、基本シール、タイムライン閲覧）
- **プレミアムプラン（月額300〜500円）**: プレミアム限定のシール・キャラクター、詳細な統計レポート、広告非表示など。

## 9. 開発スコープ

### 9.1. MVPに含むもの

- 上記「6. 機能要件」で定義された4機能の実装。
- データベース設計およびAPI開発。
- Expo (React Native) を用いたiOSおよびAndroidアプリのビルド。

### 9.2. MVPに含まないもの

- 収益化機能（アプリ内課金）
- リマインダー、タイマー通知機能
- キャラクターによるトイレ促進演出
- 詳細な統計レポート機能
- 保護者向けコラム
- 複数アカウント管理機能


## 10. 今後の展望

MVPリリース後、ユーザーのフィードバックを基に以下の機能追加を検討する。

- 統計レポート機能の強化
- ご褒美コンテンツ（キャラクター、アニメーション）の拡充
- プレミアムプランの導入
